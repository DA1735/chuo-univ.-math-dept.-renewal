# Plesk 環境向け 開発・デプロイ・運用保守マニュアル

このドキュメントは、新人エンジニア向けに本プロジェクト（React SPA + Hono APIサーバー）の「ローカル開発」「ビルドとテスト」「本番へのデプロイ」までの流れをまとめたものです。

---

## 🏗️ アプリケーションの構成について

本プロジェクトは以下の構成で動いています。

*   **フロントエンド**: React (Viteでビルド)
*   **バックエンドAPI**: Hono (TypeScriptで書かれていますが、最終的に1つのJSファイルにまとまります)
*   **データベース**: MariaDB (ローカルではDocker、本番ではPleskの機能を利用)

Plesk環境では、**1つのNode.jsプロセス（`server.js`）** が、APIの処理（`/api/*`）とフロントエンドのページ配信（それ以外のURL）の両方を担当します。

### 💡 本番環境のURL設定について
ソースコードの中に本番ドメイン（例: `https://math.v.chuo-u.ac.jp/`）をハードコードする必要は**一切ありません**。ビルド後は自動で「相対パス」と「HashRouter」により動作するため、**書き換えゼロ**で本番環境で動きます。

## 👨‍💻 1. ローカル開発環境の構築と起動

日々のソースコード修正やデザイン確認は、手元のPC（ローカル環境）で行います。

### 1.1. 前提条件と依存パッケージのインストール
*   Node.js (v18以降)、npm (または pnpm)、Docker Desktopがインストールされていること。

ターミナルを開き、プロジェクトのルートディレクトリで以下を実行します：
```bash
npm install  # （または pnpm install）
```

### 1.2. データベース（Docker）の起動とデータ投入
ローカルでの動作確認用に、DockerでMariaDBを立ち上げます。
```bash
# Dockerコンテナ（MariaDB）をバックグラウンドで起動
docker compose up -d

# テーブルの作成と、data/*.ts からの初期データ投入
npm run db:setup
npm run db:seed
```

### 1.3. 開発用サーバーの起動
ローカルでは、フロントエンドとバックエンドの2つのサーバーを別々に立ち上げます。

1.  **バックエンドAPI（Hono）の起動**：ターミナルを開き以下のコマンドを実行。
    ```bash
    npm run dev:server
    ```
    *(ポート 3001 で起動します)*
2.  **フロントエンド（React/Vite）の起動**：別のターミナルタブを開き、以下のコマンドを実行。
    ```bash
    npm run dev
    ```
    *(ポート 3000 で起動します)*

ブラウザで `http://localhost:3000` にアクセスし、サイトが表示されること・API経由でニュースやセミナーが表示されていることを確認して開発を進めてください。

## 🛠️ 2. ビルドとローカル動作確認

ソースコードの修正が終わったら、本番環境へアップロードするための「ビルド」を行います。

### 2.1. 本番用ファイルのビルド
ターミナルで以下のコマンドを実行します。
```bash
npm run build
```
ビルドが成功すると、本番環境に必要なすべてのファイルが `build/` フォルダ内に自動でまとめられます。
*   `build/dist/` （フロントエンドの全ファイル）
*   `build/server.js`（バックエンドのサーバープログラム）
*   `build/package.json`
*   `build/package-lock.json` (Plesk(npm)向けに自動変換されたロックファイル)

### 2.2. ビルド成果物のローカルテスト（本番前チェック）
本番にアップロードする前に、生成された `build/` フォルダのファイルだけで正しく動くか、ローカルでテストします。
```bash
node build/server.js
```
*(※ `Server is running on port 3001` と表示されます)*

ブラウザを開き、 `http://localhost:3001/` にアクセスします。
正しくページが表示され、データも読み込まれていれば、**ビルドは完璧に成功しています**。ターミナルで `Ctrl + C` を押してサーバーを停止してください。

## 🚀 3. Plesk 本番サーバーへのデプロイ

| 作業 | 初回 | 2回目以降（コード修正時） |
|------|:----:|:----:|
| 3.1. DB移行（SQLインポート） | ✅ 必要 | ❌ 不要（データはphpMyAdminで管理） |
| 3.2. ファイルのアップロード | ✅ 必要 | ✅ 必要（ビルドし直したファイルを上書き） |
| 3.3. Node.js設定（環境変数等） | ✅ 必要 | ❌ 不要（設定済み） |
| 3.3. 「NPMインストール」ボタン | ✅ 必要 | ⚠️ 依存パッケージが変わった場合のみ |
| 3.3. 「再起動」ボタン | ✅ 必要 | ✅ 必要（アップロード後に必ず押す） |

### 3.1. データベースの初回移行（初回のみ）
現在手元のローカルDBにある全てのデータは、プロジェクト直下の `database_init.sql` に書き出されています。これをPlesk環境へインポートします。
1.  Pleskの管理画面から対象ドメインの **「データベース」** を開き、**「phpMyAdmin」** をクリック。
2.  上部メニュー **「インポート」** タブから `database_init.sql` を選択して実行します。
> 💡 今後のデータ追加・修正について： ソースコード上の `data/*.ts` は使わず、**Plesk上の phpMyAdmin から直接ブラウザを使ってデータを追加・修正** してください。

### 3.2. ファイルのアップロード
FTPソフト等を使用して、Pleskのドキュメントルート（例: `httpdocs/`）へ、以下のファイルやフォルダを直接アップロード（配置）します。
※ `dist` は中身だけ出さないように気を付けてください。`httpdocs/dist/` となるように「フォルダごと」アップロードします。

*   ✅ **`dist/` フォルダ （フォルダごとそのまま）**
*   ✅ `server.js`
*   ✅ `package.json`
*   ✅ `pnpm-lock.yaml` （Plesk環境の pnpm install 用）

#### 📝 アップロード後のディレクトリ構成イメージ
以下のような階層になっていれば正解です。
```text
httpdocs/ <--- ドキュメントルート
  ├── dist/                # フォルダごと配置する
  │   ├── assets/
  │   ├── images/
  │   ├── pdf/
  │   └── index.html
  ├── package.json
  ├── pnpm-lock.yaml       # 重要：pnpm install用
  └── server.js
```

### 3.3. Pleskでの Node.js 設定と起動
1.  Pleskのドメイン管理画面から **「Node.js」** を開きます。
2.  設定項目を以下のように指定します：
    *   **Node.js バージョン**: 18.x 以上
    *   **アプリケーションルート**: 先ほどアップロードしたディレクトリ
    *   **アプリケーションの起動ファイル**: `server.js`
3.  **「環境変数の指定」**：データベースに接続するために以下の5つの変数を追加します。
    *   `DB_HOST` : `localhost` （Plesk内のMariaDB）
    *   `DB_PORT` : `3306` （デフォルト）
    *   `DB_USER` : 作成したDBユーザー名
    *   `DB_PASSWORD` : 作成したDBパスワード
    *   `DB_NAME` : 作成したデータベース名
4.  画面上の **「NPM インストール」** ボタンをクリック。
    *   ⚠️ Pleskの「NPMインストール」ボタンは内部で `npm install` を実行しますが、`pnpm-lock.yaml` があるため pnpm でインストールする必要があります。
    *   Pleskの画面で **「NPM インストール」** の代わりに **SSH** からコマンドを実行する場合：
    ```bash
    cd /var/www/vhosts/math.v.chuo-u.ac.jp/httpdocs
    npm install -g pnpm  # pnpm が入っていない場合のみ
    pnpm install --frozen-lockfile --prod
    ```
5.  画面上の **「アプリケーションの再起動」** ボタンをクリック。

これで本番環境へのデプロイは完了です！本番URLにアクセスして動作を確認してください。

---

## 🆘 トラブルシューティング

*   **画面に何も表示されない / 502 Bad Gateway**
    *   PleskのNode.js設定で「起動ファイル」が `server.js` になっているか確認。
    *   「NPMインストール」が失敗していないか確認して再度「再起動」。
*   **データの取得に失敗する**
    *   Pleskの環境変数（DB_USERやDB_PASSWORD等）でDB接続情報が間違っていないか確認。
    *   phpMyAdminでテーブルやデータが存在するか確認。
